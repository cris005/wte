Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

Authenticate requests to this API's endpoints by sending an Authorization header with the value "Bearer {YOUR_ACCESS_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your Access Token by performing an OAuth 2.0 authentication at the registered Authentication Server. Please contact your system administrator for more information.

Endpoints

Execute the fund movements (credits and debits) required to complete the transaction associated to the provided Ref. No.

requires authentication

Example request:
curl --request POST \
    "http://localhost/api/v1/rest/transfer" \
    --header "Authorization: Bearer {YOUR_ACCESS_TOKEN}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"Source\": 10010586384,
    \"Target\": 10010645288,
    \"Amount\": 245.5,
    \"Fee\": 10,
    \"Accounts\": [
        60001
    ]
}"

const url = new URL(
    "http://localhost/api/v1/rest/transfer"
);

const headers = {
    "Authorization": "Bearer {YOUR_ACCESS_TOKEN}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "Source": 10010586384,
    "Target": 10010645288,
    "Amount": 245.5,
    "Fee": 10,
    "Accounts": [
        60001
    ]
};

fetch(url, {
    method: "POST",

$client = new \GuzzleHttp\Client();

            'Amount' => 245.5,

$body = $response->getBody();
print_r(json_decode((string) $body));

Example: application/json

Body Parameters

Amount   number   

The amount to be transferred. Example: 245.5

Fee   number  optional  

The transaction's total fee (will be sent to the default Fee Account). If there are multiple fee recipients, leave this blank and fill out the "Accounts" parameter. Example: 10

Accounts   number[]   

The Fee Accounts and respective fees for each account. Example: {"0060001": 15.2, "0070001": 12.8}.
